为什么是叙溯

中文排版优先

Noto Serif SC 衬线标题，正文两端对齐，行高 1.8，虚线分割。不是翻译过来的排版，是从中文阅读习惯出发设计的。

双主题，零等待

朱墨与溯光内置于同一份构建产物。读者在 URL 后加 ?theme=bookish 就能换一套风格，不需要你重新构建部署。

一套代码，两种身份

siteMode: product 是产品官方站，siteMode: blog 是个人博客。同一份引擎代码，改一行配置即可。

双主题引擎

朱墨与溯光两套主题内置于同一个构建产物，构建时选定默认，运行时通过 URL 参数即刻切换——无需重新构建，无需服务器。

现代中式书卷气

朱墨

朱砂红点缀、米白墨色底、卡片式文章列表，暗色模式适配

朱砂红卡片列表衬线标题暗色适配

禅意社论风

溯光

琥珀暖色、时间轴溯源长河、正文两端对齐、虚线分割、纸质感

琥珀暖色时间轴两端对齐虚线分割

访问 ?theme=bookish 或 ?theme=zen-editorial 即刻切换

引擎特性

📝

Markdown 原生

content/ 目录下写 .md 文件，gray-matter 解析元信息，remark 渲染。无需数据库，文件即文章。

⚡

双模架构

静态模式导出纯 HTML，一键部署 GitHub Pages。动态模式接入数据库后支持在线编辑，同一套代码按需切换。

🌗

暗色模式适配

跟随系统或手动切换，朱墨与溯光各自拥有独立的暗色调配方案，夜间阅读柔和不刺眼。

🚀

零运行时依赖

纯 Next.js 静态导出，产物是一堆 HTML + JS + CSS 文件。扔到任何静态服务器上直接跑。

两种用法

siteMode: product

产品官方站

Hero 页 + 特性展示 + 主题画廊。作为博客引擎的对外展示页，像 Hexo / Hugo 官网一样。

siteMode: blog

个人博客站

双主题文章列表首页，运行时一键切换朱墨与溯光。写作、发布、切换主题，全在 Markdown 中完成。

同一份代码，修改一行配置即可切换。一套引擎，两种身份。
