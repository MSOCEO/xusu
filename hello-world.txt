16:T136d,<h2>为什么要自己造轮子</h2>
<p>2026 年 6 月，我把一个叫「叙溯」的博客引擎部署到了 GitHub Pages 上。</p>
<p>听起来像是一个深夜冲动。确实如此。但冲动背后有更久远的伏笔——我写博客写过很多轮了。WordPress 太重，Hugo 太模板化，Hexo 用着用着就搁了。每一个都不够趁手：不是功能太多让人分心，就是改个样式像在做手术。</p>
<p>我想要的东西很简单：<strong>一个为书写而生的极简引擎</strong>。Markdown 写文章，静态生成，一键部署，不改一行配置就能拥有高完成度的排版。</p>
<p>市面上没有。那就自己来。</p>
<hr>
<h2>名字的故事</h2>
<p>「叙溯」这个名字想了很久。</p>
<p><strong>叙</strong>，是讲述。写博客说到底就是把脑子里纠缠的想法，逐字逐句摊开，熨平，交给读者。</p>
<p><strong>溯</strong>，是回溯。每一篇文章都是时间的切片——写下的那一刻你是什么状态，想了什么，焦虑了什么，兴奋了什么。回头看，就是一条清晰可见的成长轨迹。</p>
<p>合在一起，「叙溯」的意思是：<strong>一边讲述，一边溯源。在书写中厘清思路，在回顾中看见轨迹。</strong></p>
<p>这不是一个功能列表里的项目名。这是一个写作信念的宣言。</p>
<hr>
<h2>两套主题，两种心境</h2>
<p>博客引擎的骨架搭好之后，我开始打磨样式。在我看来，主题不是「皮肤」，而是<strong>阅读情绪的容器</strong>。不同的内容，应该配上不同的氛围。</p>
<p>由此诞生了两套主题：</p>
<h3>「朱墨」—— 现代中式书卷气</h3>
<p>主色调是朱砂红 <code>oklch(0.55 0.22 25)</code>，底是米白与墨色。标题用 Noto Serif SC 衬线体，正文用系统无衬线。</p>
<p>首页是标题区加卡片式文章列表。每篇文章是一张卡片，hover 时底色从米白过渡到极浅的朱砂暖调，像翻动一页宣纸。页脚的「叙说与溯源」竖排，融进留白边缘。</p>
<p>适合长文、散文、技术思考。安静，温润，有书卷气。</p>
<h3>「溯光」—— 禅意社论风</h3>
<p>琥珀暖色系（<code>#8E5529</code> / <code>#D48F59</code>），正文行高拉到 1.8，两端对齐，Blockquote 是 4px 粗线。</p>
<p>首页不是卡片——是一条<strong>时间轴</strong>。每篇文章左侧一条细线，hover 时圆点从琥珀色过渡到暖金，右面是标题加日期。像翻阅年鉴。</p>
<p>文章页的超大衬线标题和引号斜体摘要，把「阅读」这件事拉回到了纸媒黄金年代的社论体感。页脚标语是一句简短的提醒：<strong>一边讲述，一边溯源。</strong></p>
<p>适合日记、年度回顾、或任何需要沉淀后再说的话。</p>
<p>两套主题可以<strong>运行时切换</strong>——在任何页面 URL 后面加 <code>?theme=bookish</code> 或 <code>?theme=zen-editorial</code> 就能立刻切换，无需重新构建，无需刷新服务器。构建产物中两套 HTML 同时存在，CSS 负责显隐。</p>
<hr>
<h2>技术上的几件事</h2>
<p>技术细节我不打算展开太多，但有几件值得提的事：</p>
<p><strong>架构上</strong>，用 Next.js 的静态导出（<code>next export</code>）跑在 GitHub Pages 上。文章存在 <code>content/</code> 目录的 <code>.md</code> 文件里，gray-matter 解析 front matter，remark 渲染 Markdown。不依赖数据库，不依赖运行时。</p>
<p><strong>主题系统</strong>，两套首页作为独立组件共存于构建产物中，通过 CSS <code>[data-theme]</code> 属性切换。文章页面的差异化样式也用条件渲染处理，确保切换后内容完全不丢失。</p>
<p><strong>部署上</strong>，GitHub Pages 有一些坑：<code>.nojekyll</code> 必须加，<code>spa.html</code> 的 404 回退要写对，CDN 缓存刷新要主动触发，Git Tree API 推送的时候 <code>.gitattributes</code> 会影响文件编码。这些细节，每一个都花去了一两个小时调试。</p>
<p>但踩完坑之后，剩下的就是纯粹的快乐：改一行 Markdown，<code>git push</code>，五分钟后新文章上线。</p>
<hr>
<h2>接下来</h2>
<p>「叙溯」还有很多事要做：</p>
<ul>
<li>完善动态模式下的管理后台</li>
<li>增加 RSS 订阅</li>
<li>优化移动端阅读体验</li>
<li>更多主题，更多「阅读情绪的容器」</li>
</ul>
<p>但最重要的，是<strong>开始写</strong>。引擎只是载体，字才是主语。</p>
<p>这是一篇创刊文，也是一个起点。如果你也在找一个趁手的写作工具，不妨看看<a href="https://github.com/msoceo/xusu">叙溯的仓库</a>。如果你只是路过，那么希望这两套主题能给你带来五分钟安静的阅读时间。</p>
<hr>
<p><em>在书写中厘清思路，在回顾中看见轨迹。</em></p>
