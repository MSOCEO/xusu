## 为什么要自己造轮子

2026 年 6 月，我把一个叫「叙溯」的博客引擎部署到了 GitHub Pages 上。

听起来像是一个深夜冲动。确实如此。但冲动背后有更久远的伏笔——我写博客写过很多轮了。WordPress 太重，Hugo 太模板化，Hexo 用着用着就搁了。每一个都不够趁手：不是功能太多让人分心，就是改个样式像在做手术。

我想要的东西很简单：**一个为书写而生的极简引擎**。Markdown 写文章，静态生成，一键部署，不改一行配置就能拥有高完成度的排版。

市面上没有。那就自己来。

---

## 名字的故事

「叙溯」这个名字想了很久。

**叙**，是讲述。写博客说到底就是把脑子里纠缠的想法，逐字逐句摊开，熨平，交给读者。

**溯**，是回溯。每一篇文章都是时间的切片——写下的那一刻你是什么状态，想了什么，焦虑了什么，兴奋了什么。回头看，就是一条清晰可见的成长轨迹。

合在一起，「叙溯」的意思是：**一边讲述，一边溯源。在书写中厘清思路，在回顾中看见轨迹。**

这不是一个功能列表里的项目名。这是一个写作信念的宣言。

---

## 两套主题，两种心境

博客引擎的骨架搭好之后，我开始打磨样式。在我看来，主题不是「皮肤」，而是**阅读情绪的容器**。不同的内容，应该配上不同的氛围。

由此诞生了两套主题：

### 「朱墨」—— 现代中式书卷气

主色调是朱砂红 `oklch(0.55 0.22 25)`，底是米白与墨色。标题用 Noto Serif SC 衬线体，正文用系统无衬线。

首页是标题区加卡片式文章列表。每篇文章是一张卡片，hover 时底色从米白过渡到极浅的朱砂暖调，像翻动一页宣纸。页脚的「叙说与溯源」竖排，融进留白边缘。

适合长文、散文、技术思考。安静，温润，有书卷气。

### 「溯光」—— 禅意社论风

琥珀暖色系（`#8E5529` / `#D48F59`），正文行高拉到 1.8，两端对齐，Blockquote 是 4px 粗线。

首页不是卡片——是一条**时间轴**。每篇文章左侧一条细线，hover 时圆点从琥珀色过渡到暖金，右面是标题加日期。像翻阅年鉴。

文章页的超大衬线标题和引号斜体摘要，把「阅读」这件事拉回到了纸媒黄金年代的社论体感。页脚标语是一句简短的提醒：**一边讲述，一边溯源。**

适合日记、年度回顾、或任何需要沉淀后再说的话。

两套主题可以**运行时切换**——在任何页面 URL 后面加 `?theme=bookish` 或 `?theme=zen-editorial` 就能立刻切换，无需重新构建，无需刷新服务器。构建产物中两套 HTML 同时存在，CSS 负责显隐。

## 技术上的几件事

技术细节我不打算展开太多，但有几件值得提的事：

**架构上**，用 Next.js 的静态导出（`next export`）跑在 GitHub Pages 上。文章存在 `content/` 目录的 `.md` 文件里，gray-matter 解析 front matter，remark 渲染 Markdown。不依赖数据库，不依赖运行时。

**主题系统**，两套首页作为独立组件共存于构建产物中，通过 CSS `[data-theme]` 属性切换。文章页面的差异化样式也用条件渲染处理，确保切换后内容完全不丢失。

**部署上**，GitHub Pages 有一些坑：`.nojekyll` 必须加，`spa.html` 的 404 回退要写对，CDN 缓存刷新要主动触发，Git Tree API 推送的时候 `.gitattributes` 会影响文件编码。这些细节，每一个都花去了一两个小时调试。

但踩完坑之后，剩下的就是纯粹的快乐：改一行 Markdown，`git push`，五分钟后新文章上线。

## 接下来

「叙溯」还有很多事要做：

- 完善动态模式下的管理后台
- 增加 RSS 订阅
- 优化移动端阅读体验
- 更多主题，更多「阅读情绪的容器」

但最重要的，是**开始写**。引擎只是载体，字才是主语。

这是一篇创刊文，也是一个起点。如果你也在找一个趁手的写作工具，不妨看看[叙溯的仓库](https://github.com/msoceo/xusu)。如果你只是路过，那么希望这两套主题能给你带来五分钟安静的阅读时间。
